What you're building
And why it's structured this way
🎨 A two-layer token system

Global tokens hold raw values (your full color palette, every spacing step). Semantic tokens reference globals and give values meaning — color/brand/default points to color/purple/600. Components only ever use semantic tokens.

🌗 Two modes, one source

Light and dark mode are handled through Figma Variable Modes — not duplicate components. You define the mode values once; Figma (and later your CSS) swaps them automatically.

🔌 Tokens Studio as the bridge

Tokens Studio reads your Figma Variables and exports a structured tokens.json file. That file is what feeds Style Dictionary in the code project. Your Figma decisions become CSS custom properties.

🧩 Four components, token-driven

Button, Card, Input, and Badge. Every color, spacing, and radius value in these components must reference a variable — no hardcoded values anywhere in the file.

Figma Variables
Build the two-layer token system in the Variables panel
⚠️

Naming is everything. Tokens Studio maps your Figma Variable names directly to JSON keys. Use forward-slash grouping: color/brand/default not Brand Default Color. Wrong naming = broken export.

Layer 1 — Global collection Required

Raw values only. No references to other variables. This is your palette and scale — nothing here has a semantic meaning yet.

A

Create the Global collection first

Open Variables panel → + New collection → name it "Global". Add all raw values. Do not apply these to any component directly — they're the palette only.

B

Create the Semantic collection and set references

+ New collection → name it "Semantic". For each variable, click the value field and use the library icon to point it at a Global variable — not a raw color. If you're typing a hex code here, you're doing it wrong.

C

Add Modes to the Semantic collection

In the Semantic collection header, click "1 mode" → add a second mode named "Dark". Override each variable's dark value — point to darker global stops for surfaces, lighter stops for text. Test by switching modes on a frame.

D

Document on the Tokens page

Build an auto-layout reference frame showing each semantic token, its value in light mode, and its value in dark mode. Use text + color fill swatches. This is part of your grade.

4
Light & dark mode preview
Prove your tokens work by switching modes on assembled screens
1

Assemble a preview screen on the ☀️ Light preview page

Create a 390×844 frame (iPhone 14 size). Drop in instances of all four components in realistic arrangements — a form with Input + Button, a list of Cards with Badges. This is not a full design; it's a token stress test.

2

Duplicate the frame to the 🌙 Dark preview page

Select the frame → right-click → Move to page → Dark preview. Do not re-design anything.

3

Switch the mode on the dark frame

Select the frame → in the right panel under Variables, change the Semantic collection mode from Light to Dark. Every component should update automatically. If any color doesn't change, that component has a hardcoded value — find and fix it.

4

Side-by-side screenshot on your Cover page

Place a cropped screenshot of both frames on your Cover page to demonstrate the mode switch works.

⚠️

Common failure: Components built with "detached" instances or hardcoded fills won't respond to mode changes. If your dark mode doesn't work, inspect each layer — click the fill and check whether it shows a variable name or a hex value.

5
Tokens Studio export
Bridge your Figma variables to the code project
🔌

Install from the Figma Community: search "Tokens Studio for Figma" by Jan Six. It's free for the features you need here.

1

Open Tokens Studio → Import Variables

In the plugin panel, click the settings gear → Import Variables from Figma. It will read your Global and Semantic collections and generate a matching token set.

2

Review the token structure in the plugin

You should see two sets: Global and Semantic. Expand a few entries and verify the names match the color/brand/default convention — forward slashes become nested JSON keys. Fix any naming inconsistencies now before export.

3

Export to JSON

Click the export icon (↑) → Export to file → save as tokens.json. This is the file you'll drop into src/tokens/ in the Pebble UI code project for Part 2.

4

Screenshot the plugin panel for your Tokens Studio page

Take a screenshot showing the imported token tree in the plugin. Paste it on the 🔌 Tokens Studio page in your Figma file as documentation.

✅

Verify the export worked: Open tokens.json in a text editor. You should see nested objects like "color": { "brand": { "default": { "value": "{color.purple.600}" } } }. If you see raw hex values instead of references, the semantic layer isn't set up correctly in Figma.
